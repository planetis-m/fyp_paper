<chunk page=402 label="Bayesian Networks Characteristics">
Bayesian networks provide a powerful approach for representing probabilistic relationships between attributes and class labels with the help of graphical models. They are able to capture complex forms of dependencies among variables. Apart from encoding prior beliefs, they are also able to model the presence of latent (unobserved) factors as hidden variables in the graph. Bayesian networks are thus quite expressive and provide predictive as well as descriptive insights about the behavior of attributes and class labels.

<chunk page=402 label="Bayesian Networks Characteristics">
Bayesian networks can easily handle the presence of correlated or redundant attributes, as opposed to the naive Bayes classifier. This is because Bayesian networks do not use the naive Bayes assumption about conditional independence, but instead are able to express richer forms of conditional independence. Similar to the naive Bayes classifier, Bayesian networks are also quite robust to the presence of noise in the training data. Further, they can handle missing values during training as well as testing. If a test instance contains an attribute with a missing value, then a Bayesian network can perform inference by treating that attribute as an unobserved node.

<chunk page=403 label="Bayesian Networks Characteristics">
Bayesian networks are well-suited for handling incompleteness in the data and can work with partial information. However, unless the pattern with which missing values occurs is completely random, then their presence will likely introduce some degree of error and/or bias into the analysis. Bayesian networks are robust to irrelevant attributes that contain no discriminatory information about the class labels. Such attributes show no impact on the conditional probability of the target class, and are thus rightfully ignored.

<chunk page=403 label="Bayesian Networks Characteristics">
Learning the structure of a Bayesian network is a cumbersome task that often requires assistance from expert knowledge. However, once the structure has been decided, learning the parameters of the network can be quite straightforward, especially if all the variables in the network are observed. Due to its additional ability of representing complex forms of relationships, Bayesian networks are more susceptible to overfitting as compared to the naive Bayes classifier. Furthermore, Bayesian networks typically require more training instances for effectively learning the probability tables than the naive Bayes classifier. Although the sum-product algorithm provides computationally efficient techniques for performing inference over tree-like graphs, the complexity of the approach increases significantly when dealing with generic graphs of large sizes. In situations where exact inference is computationally infeasible, it is quite common to use approximate inference techniques.

<chunk page=404 label="Logistic Regression">
Logistic regression is a probabilistic discriminative model that directly estimates the odds of a data instance using its attribute values. The naive Bayes and Bayesian network classifiers provide different ways of estimating the conditional probability of an instance x given class y, P(x|y). Such models are known as probabilistic generative models. However, for making predictions, we are interested in computing the posterior probability P(y|x). The ratio P(y=1|x)/P(y=0|x) is known as the odds. If this ratio is greater than 1, then x is classified as y=1. Otherwise, it is assigned to class y=0. Classification models that directly assign class labels without computing class-conditional probabilities are called discriminative models.

<chunk page=404 label="Logistic Regression">
The basic idea of logistic regression is to use a linear predictor, z = w^T x + b, for representing the odds of x as follows: P(y=1|x)/P(y=0|x) = e^z = e^{w^T x + b}, where w and b are the parameters of the model. If w^T x + b > 0, then x belongs to class 1 since its odds is greater than 1. Since P(y=0|x) + P(y=1|x) = 1, we can express P(y=1|x) as a function of z: P(y=1|x) = 1/(1 + e^{-z}) = sigma(z), where the function sigma is known as the logistic or sigmoid function. We can see that sigma(z) >= 0.5 only when z >= 0. P(y=0|x) = 1 - sigma(z) = 1/(1 + e^z).

<chunk page=406 label="Logistic Regression GLM">
Logistic regression belongs to a broader family of statistical regression models, known as generalized linear models (GLM). In these models, the target variable y is considered to be generated from a probability distribution P(y|x), whose mean mu can be estimated using a link function g: g(mu) = z = w^T x + b. For binary classification using logistic regression, y follows a Bernoulli distribution (y can be either 0 or 1) and mu is equal to P(y=1|x). The link function g of logistic regression, called the logit function, can be represented as g(mu) = log(mu/(1 - mu)). Depending on the choice of link function g and the form of probability distribution P(y|x), GLMs are able to represent a broad family of regression models, such as linear regression and Poisson regression.

<chunk page=407 label="Logistic Regression MLE">
The parameters of logistic regression, (w, b), are estimated during training using maximum likelihood estimation (MLE). This involves computing the likelihood of observing the training data given (w, b) and determining the model parameters that yield maximum likelihood. For a given training instance x_i, the likelihood of observing y_i given x_i, w, and b is expressed as: P(y_i|x_i, w, b) = (sigma(w^T x_i + b))^{y_i} * (1 - sigma(w^T x_i + b))^{1 - y_i}. The likelihood of all training instances L(w, b) is computed by taking the product of individual likelihoods assuming independence among training instances.

<chunk page=408 label="Logistic Regression Cross Entropy">
Since the naive computation of likelihood can become numerically unstable when n is large, a more practical approach is to consider the negative logarithm of the likelihood function, also known as the cross entropy function. The cross entropy is a loss function that measures how unlikely it is for the training data to be generated from the logistic regression model with parameters (w, b). We want to find model parameters that result in the lowest cross entropy. The loss function E(w, b) = -log L(w, b) is a convex function, meaning any minima will be a global minima. Hence, we can use standard convex optimization techniques such as the Newton-Raphson method.

<chunk page=409 label="Logistic Regression Newton-Raphson">
The Newton-Raphson method is an iterative method for finding optimal parameters that uses the following update equation at every iteration: w^{(new)} = w^{(old)} - H^{-1} * gradient E(w), where the gradient is the first-order derivative and H is the second-order derivative (Hessian) of the loss function. The first-order derivative simplifies to: gradient E(w) = X^T (sigma - y). The second-order derivative simplifies to: H = X^T R X, where R is a diagonal matrix whose i-th diagonal element R_{ii} = sigma_i(1 - sigma_i). The update equation at the k-th iteration becomes: w^{(k+1)} = w^{(k)} - (X^T R_k X)^{-1} X^T (sigma_k - y).

<chunk page=410 label="Logistic Regression Characteristics">
Logistic regression is a discriminative model for classification that directly computes the posterior probabilities without making any assumption about the class-conditional probabilities. It can be easily extended to multiclass classification, where it is known as multinomial logistic regression. However, its expressive power is limited to learning only linear decision boundaries. The learned parameters can be analyzed to understand the relationships between attributes and class labels. Because logistic regression does not involve computing densities and distances in the attribute space, it can work more robustly even in high-dimensional settings than distance-based methods. However, the objective function does not involve any term relating to model complexity, so logistic regression does not provide a way to make a trade-off between model complexity and training performance as compared to other models such as SVM.

<chunk page=411 label="Logistic Regression Regularization">
Variants of logistic regression can easily be developed to account for model complexity by including appropriate regularization terms in the objective function along with the cross entropy function. Logistic regression can handle irrelevant attributes by learning weight parameters close to 0 for attributes that do not provide any gain in performance. Since it uses a linear combination of attribute values, it is naturally robust to irrelevant attributes. However, it cannot naturally handle redundant attributes without careful regularization, since the model may assign arbitrary weights to redundant attributes without any change in performance. Similar to Bayesian models, logistic regression is quite robust to noise in the attribute values.

<chunk page=412 label="Rule-Based Classifier">
A rule-based classifier uses a collection of "if...then..." rules to classify data instances. A rule r is expressed as: (condition) -> y, where the left-hand side (antecedent) is a conjunction of conditions on attributes and the right-hand side (consequent) is the predicted class label. An example rule: (Blood Type=Warm) AND (Lays Eggs=Yes) -> Bird. A rule r covers an instance x if the attributes of x satisfy the antecedent of r. The coverage of a rule is the fraction of training instances that trigger the rule. The accuracy of a rule is the fraction of triggered instances that are correctly classified.

<chunk page=413 label="Rule-Based Classifier Characteristics">
Characteristics of rule-based classifiers: (1) The rule set can be mutually exclusive (no two rules triggered by same instance) and exhaustive (every instance triggers at least one rule). Mutually exclusive and exhaustive rule sets produce exactly one rule per instance. (2) If the rule set is not mutually exclusive, an instance can trigger multiple rules with conflicting predictions, requiring conflict resolution strategies. (3) If the rule set is not exhaustive, some instances may not trigger any rule, requiring a default class strategy.

<chunk page=414 label="Rule Ordering and Measures">
When rules are not mutually exclusive, conflict resolution can use rule-based ordering (rules ranked by some quality measure) or class-based ordering (rules belonging to the same class grouped together). Rule quality measures include: coverage (fraction of instances covered by the rule), accuracy (fraction of covered instances correctly classified), and a combined measure like Laplace estimate. Rules can be generated using direct methods (extracting rules directly from data, e.g., RIPPER) or indirect methods (extracting rules from other classification models like decision trees).

<chunk page=415 label="RIPPER Algorithm">
RIPPER is a direct method for rule generation that follows a sequential covering strategy. For two-class problems, it treats one class as positive and the other as negative. Rules are learned one at a time, and instances covered by a learned rule are removed before learning the next rule. The process continues until all positive instances are covered. For multi-class problems, classes are ordered by increasing prevalence, and RIPPER learns rules for each class sequentially.

<chunk page=416 label="Rule Growing and Pruning">
Rule growing starts with an initial rule and gradually adds conjuncts to improve quality. The most common approach is to add the conjunct that improves the rule quality measure the most, continuing until the rule reaches a desired quality level. For general-to-specific growing, we start with an empty rule that covers all instances and add conjuncts to reduce coverage while improving accuracy. For specific-to-general growing, we start with a rule that covers one instance and remove conjuncts to increase coverage while maintaining accuracy. After growing, rules are pruned to improve generalization by removing conjuncts that hurt performance on a validation set.

<chunk page=417 label="Rule Pruning and Evaluation">
To prune a rule, the following metric is commonly used: v(Rule) = (p - n)/(p + n), where p is the number of positive instances covered and n is the number of negative instances covered. If v improves after removing a conjunct, that conjunct is pruned. Pruning is done from the last conjunct added (the most specific) backward. After generating and pruning the rule set, RIPPER performs an optimization phase to examine each rule and either replace it with a better rule or add new rules to improve coverage.

<chunk page=418 label="Rule Extraction from Decision Trees">
An indirect method for rule extraction builds a decision tree first and then converts it into a set of rules by tracing the path from root to each leaf. Each path becomes a rule where internal nodes form the antecedent and the leaf label becomes the consequent. The resulting rules are then pruned to remove redundant conditions. Advantages of this approach include leveraging well-established decision tree algorithms, though the rules may be more complex than necessary since decision tree splits are chosen globally rather than locally per rule.

<chunk page=419 label="Rule-Based Classifier Characteristics">
Characteristics of rule-based classifiers: (1) Rule-based classifiers are highly expressive and can represent complex decision boundaries. (2) The rule-based representation is easy to interpret and provides insights about the classification logic. (3) Rule-based classifiers can naturally handle both categorical and continuous attributes, though continuous attributes need to be discretized. (4) They are robust to irrelevant attributes since such attributes are unlikely to appear in any rule. (5) Rule generation can be computationally expensive, especially for large data sets.

<chunk page=420 label="Neural Networks Introduction">
Artificial Neural Networks (ANN) are inspired by biological neural networks in the brain. An ANN consists of a set of connected computational units called neurons (or nodes) organized in layers. A neuron takes multiple inputs, computes a weighted sum, adds a bias, and applies an activation function to produce an output. Common activation functions include the sigmoid function, hyperbolic tangent (tanh), and rectified linear unit (ReLU). ANNs can learn complex nonlinear decision boundaries and have achieved state-of-the-art performance in many domains.

<chunk page=421 label="Perceptron">
The perceptron is the simplest form of a neural network consisting of a single neuron. It computes a weighted sum of inputs plus a bias: z = w^T x + b, and applies a step (threshold) activation function to produce a binary output. The perceptron learning rule updates the weights when a misclassification occurs: w = w + eta * (y - y_hat) * x, where eta is the learning rate, y is the true label, and y_hat is the predicted label. The perceptron can only learn linearly separable functions and cannot solve problems like XOR.

<chunk page=422 label="Multilayer Neural Network">
A multilayer neural network (multilayer perceptron or MLP) consists of an input layer, one or more hidden layers, and an output layer. Each neuron in a hidden layer computes a weighted sum of its inputs and applies a nonlinear activation function. The presence of hidden layers with nonlinear activations enables the network to learn complex nonlinear decision boundaries. The number of hidden layers and neurons per layer are hyperparameters. An MLP with at least one hidden layer is a universal function approximator, meaning it can approximate any continuous function given enough neurons.

<chunk page=423 label="Neural Network Forward Propagation">
Forward propagation is the process of computing the output of the neural network given an input. For a single hidden layer network with d input features, m hidden neurons, and k output neurons: the hidden layer output is h = f(W_1 x + b_1) where f is the activation function, W_1 is the d x m weight matrix, and b_1 is the bias vector. The output layer computes o = g(W_2 h + b_2) where g is the output activation function. For binary classification, g is the sigmoid function. For multiclass classification, g is the softmax function.

<chunk page=424 label="Backpropagation">
Backpropagation is the algorithm used to train neural networks by computing the gradient of the loss function with respect to each weight. It uses the chain rule of calculus to propagate error signals backward through the network. For each weight, the gradient is computed as the product of the error signal at the downstream neuron and the activation of the upstream neuron. The weights are then updated using gradient descent: w = w - eta * gradient, where eta is the learning rate. Backpropagation enables efficient computation of gradients in multilayer networks.

<chunk page=425 label="Backpropagation Algorithm Details">
The backpropagation algorithm proceeds in two phases for each training instance. In the forward pass, inputs are propagated through the network to compute the output and loss. In the backward pass, the error signals are propagated backward to compute gradients for all weights. The key insight is that the gradient for a weight in layer l depends on the error signal from layer l+1, which is computed recursively. For a network with L layers, we first compute the error at the output layer, then propagate it back through each hidden layer. The error signal at hidden neuron j in layer l is: delta_j^l = f'(z_j^l) * sum_k(delta_k^{l+1} * w_{jk}^{l+1}).

<chunk page=426 label="Neural Network Loss Functions">
The choice of loss function depends on the task. For binary classification, the cross entropy loss is used: L = -[y log(y_hat) + (1-y) log(1-y_hat)]. For multiclass classification with k classes, the categorical cross entropy is used: L = -sum_{j=1}^{k} y_j log(y_hat_j), where y is a one-hot encoded vector. For regression, the mean squared error is commonly used: L = (1/n) sum_{i=1}^{n} (y_i - y_hat_i)^2. The overall loss is typically computed as the average over all training instances.

<chunk page=427 label="Neural Network Activation Functions">
Common activation functions for hidden layers include: (1) Sigmoid: sigma(z) = 1/(1 + e^{-z}), output range (0,1). (2) Tanh: tanh(z), output range (-1,1), zero-centered. (3) ReLU: max(0, z), computationally efficient, avoids vanishing gradient for positive inputs. (4) Leaky ReLU: max(alpha*z, z) for small alpha, avoids dead neurons. For the output layer: sigmoid for binary classification, softmax for multiclass classification (converts logits to probabilities that sum to 1), and linear (identity) for regression.

<chunk page=428 label="Neural Network Training">
Neural network training involves several considerations. Weight initialization matters: small random values prevent symmetry and ensure different neurons learn different features. Xavier/He initialization methods scale initial weights based on layer sizes. The learning rate controls step size during gradient descent; too large causes oscillation, too small causes slow convergence. Mini-batch gradient descent computes gradients on subsets of the data, providing a balance between computation speed and gradient accuracy. Momentum and adaptive learning rate methods (Adam, RMSprop) improve convergence.

<chunk page=429 label="Neural Network Overfitting">
Neural networks with many parameters are prone to overfitting. Regularization techniques include: (1) L2 regularization (weight decay): adds penalty term lambda * ||w||^2 to the loss function to discourage large weights. (2) Dropout: randomly deactivates a fraction of neurons during training, preventing co-adaptation and providing an implicit ensemble effect. (3) Early stopping: monitors validation performance and stops training when it starts degrading. (4) Data augmentation: artificially increases training data by applying transformations. (5) Batch normalization: normalizes layer inputs to stabilize and accelerate training.

<chunk page=430 label="Neural Network Hyperparameters">
Key hyperparameters in neural networks include: (1) Number of hidden layers (depth): more layers can capture more complex features but increase overfitting risk and training difficulty. (2) Number of neurons per layer (width): more neurons increase model capacity. (3) Learning rate: controls the step size in weight updates. (4) Batch size: number of instances used in each gradient computation. (5) Number of epochs: number of complete passes through the training data. (6) Regularization strength: controls the trade-off between fitting training data and model complexity.

<chunk page=431 label="ANN Characteristics">
Characteristics of artificial neural networks: (1) ANNs can model complex nonlinear relationships between attributes and class labels. (2) They have strong theoretical guarantees as universal function approximators (given enough neurons). (3) ANNs can handle both numerical and categorical attributes (categorical ones need encoding). (4) Training can be computationally expensive, especially for large networks. (5) ANNs are susceptible to overfitting, particularly when the training data is limited relative to model complexity. (6) The learned model is a black box, making it difficult to interpret the learned relationships.

<chunk page=432 label="Deep Learning Introduction">
Deep learning refers to training neural networks with multiple hidden layers. The key challenge is that deeper networks are harder to train due to vanishing/exploding gradients, where gradient signals become too small or too large as they propagate through many layers. Deep learning techniques address these challenges through careful initialization, activation function choices (ReLU helps mitigate vanishing gradients), and specialized architectures. Deep learning has enabled breakthroughs in computer vision, natural language processing, and speech recognition.

<chunk page=433 label="Vanishing and Exploding Gradients">
In deep networks, gradients computed during backpropagation can vanish (become extremely small) or explode (become extremely large) as they are propagated backward through many layers. This is because gradients are multiplied by weights and activation function derivatives at each layer, and repeated multiplication can cause exponential shrinking or growth. Vanishing gradients prevent lower layers from learning effectively, while exploding gradients cause unstable training. Solutions include: using ReLU activation (derivative is 0 or 1), batch normalization, careful weight initialization, gradient clipping, and residual connections.

<chunk page=434 label="Deep Learning Pretraining">
Pretraining is a strategy for training deep networks by learning one layer at a time. In unsupervised pretraining, each layer is trained using an autoencoder that learns to reconstruct its input. The first autoencoder learns features from raw input, the second autoencoder learns features from the first layer's representations, and so on. After pretraining all layers, the network is fine-tuned using supervised learning with backpropagation. This greedy layer-wise approach helps find good initial parameters and was historically important for training deep networks before modern techniques made end-to-end training feasible.

<chunk page=435 label="Autoencoders">
An autoencoder is an unsupervised neural network that learns to reconstruct its input. It consists of an encoder that maps input to a lower-dimensional representation (code) and a decoder that reconstructs the input from the code. The autoencoder is trained to minimize the reconstruction error. If the code dimension is smaller than the input, the autoencoder learns a compressed representation. Autoencoders can be used for dimensionality reduction, feature learning, and as building blocks for pretraining deep networks. Denoising autoencoders add noise to the input and learn to recover the original, producing more robust features.

<chunk page=436 label="Deep Learning Regularization">
Regularization techniques specific to deep learning include: (1) Dropout: during training, randomly set a fraction of neuron outputs to zero. This prevents neurons from co-adapting and forces the network to learn redundant representations. At test time, all neurons are used but their outputs are scaled. (2) DropConnect: a generalization of dropout that randomly drops weights instead of neuron activations. (3) These methods effectively create an ensemble of many thinned networks, improving generalization. The dropout rate is a hyperparameter, typically set between 0.2 and 0.5.

<chunk page=437 label="Deep Learning Pretraining Strategies">
Pretraining strategies for deep networks include: (1) Supervised pretraining: trains the network one layer at a time using labeled data. Each new layer is added and trained while keeping previous layers frozen or jointly fine-tuned. (2) Unsupervised pretraining: uses autoencoders to learn feature representations layer by layer without labels, then fine-tunes the full network with supervision. (3) The choice between supervised and unsupervised pretraining depends on the availability of labeled data. Unsupervised pretraining is useful when labeled data is scarce.

<chunk page=438 label="Deep Learning Supervised Pretraining">
In supervised pretraining, the network is built incrementally. For a network with h hidden layers: first train a network with one hidden layer, then freeze the first hidden layer and add a second hidden layer, train the second layer, and so on. Each time a new hidden layer is added, only the new layer's parameters are updated while the previously learned parameters remain fixed. This gradual approach helps avoid poor local minima and can lead to better solutions than training all layers simultaneously from random initialization, especially for very deep networks.

<chunk page=439 label="Autoencoder Pretraining">
In autoencoder pretraining, each hidden layer is pretrained using an autoencoder. The first autoencoder takes raw input and learns to reconstruct it through a bottleneck (the first hidden layer). The encoder part of the first autoencoder becomes the first hidden layer of the deep network. The second autoencoder takes the first hidden layer's output as input and learns to reconstruct it through the second hidden layer, and so on. After all layers are pretrained, the full network is assembled and fine-tuned using supervised backpropagation. This approach helps initialize the network weights in a good region of the parameter space.

<chunk page=440 label="Denoising Autoencoders">
A denoising autoencoder is trained to recover the original input from a corrupted version. During training, noise is added to the input (e.g., randomly setting some features to zero), and the autoencoder learns to reconstruct the clean input. This forces the autoencoder to learn more robust features that capture the underlying structure of the data rather than simply memorizing the input. The corruption level (fraction of features corrupted) is a hyperparameter. Denoising autoencoders are particularly useful when the input data is noisy or when learning features that are invariant to small perturbations.

<chunk page=441 label="Fine-Tuning Deep Networks">
After pretraining, the deep network is fine-tuned using supervised learning. All layers are unfrozen and the entire network is trained end-to-end using backpropagation. The pretrained weights serve as a much better initialization than random weights, leading to faster convergence and often better local minima. Fine-tuning can be done by minimizing the cross entropy loss for classification tasks. The learning rate during fine-tuning is typically smaller than during pretraining to avoid drastically changing the pretrained representations.

<chunk page=442 label="Deep Learning Joint Training">
An alternative to sequential pretraining is joint training, where multiple objectives are optimized simultaneously. For example, at each layer during pretraining, we can jointly minimize both an unsupervised reconstruction loss and a supervised classification loss. This combined approach balances the objectives of representing the input data well and improving classification performance. The trade-off between the two objectives is controlled by a hyperparameter alpha. Joint training often shows better generalization performance than either purely supervised or purely unsupervised pretraining alone.

<chunk page=443 label="Deep Learning Practical Considerations">
Practical considerations for training deep networks: (1) GPU acceleration: deep learning benefits enormously from parallel computation on GPUs. (2) Large datasets: deep networks typically require large amounts of data to achieve good performance. (3) Hyperparameter tuning: the number of hyperparameters grows with network depth, making systematic search important. (4) Monitoring training: track training and validation loss to detect overfitting and guide early stopping. (5) Data preprocessing: normalizing input features to zero mean and unit variance improves training stability.

<chunk page=444 label="Deep Learning Techniques Summary">
Summary of deep learning techniques for addressing training challenges: (1) For vanishing/exploding gradients: use ReLU activations, batch normalization, careful initialization, residual connections. (2) For overfitting: use dropout, L2 regularization, early stopping, data augmentation. (3) For poor local minima: use pretraining strategies (unsupervised or supervised), momentum-based optimization, adaptive learning rates. (4) For computational efficiency: use mini-batch training, GPU acceleration. These techniques work together to enable successful training of very deep networks.

<chunk page=445 label="Deep Learning Applications">
Deep learning has found applications in numerous domains: (1) Computer vision: image classification, object detection, facial recognition, using Convolutional Neural Networks (CNNs). (2) Natural language processing: machine translation, sentiment analysis, text generation, using Recurrent Neural Networks (RNNs) and Transformer architectures. (3) Speech recognition: converting audio to text. (4) Drug discovery: predicting molecular properties. (5) Autonomous vehicles: perception and decision-making. The success of deep learning in these domains often depends on large labeled datasets and computational resources.

<chunk page=446 label="Deep Learning Pretraining Combined">
Combined pretraining approaches use both unsupervised and supervised objectives during layer-wise training. At every run of pretraining for each hidden layer, two output layers are used: one for reconstruction (unsupervised) and one for supervised classification. The parameters of each hidden layer are learned by jointly minimizing the losses on both output layers, weighted by a trade-off hyperparameter alpha. This combined approach often shows better generalization performance than either approach alone, as it balances the competing objectives of representing the input data and improving classification performance.

<chunk page=447 label="Deep Learning Reusability">
An ANN model trained for some task can be easily re-used for a different task that involves the same attributes by using pretraining strategies. The learned parameters of the original task serve as initial parameter choices for the target task. This promotes re-usability of learning, which is useful when the target application has fewer labeled training instances. The autoencoder for pretraining can help eliminate irrelevant attributes and reduce the impact of redundant attributes by representing them as copies of the same attribute.

<chunk page=448 label="Deep Learning Characteristics">
Characteristics of deep learning: (1) Deep learning promotes re-usability of learned models across tasks. (2) Regularization techniques like dropout help reduce model complexity and promote good generalization, especially in high-dimensional settings with small labeled datasets. (3) Autoencoder pretraining can eliminate irrelevant attributes and reduce redundant attribute impact. (4) Although learning can succumb to locally optimal solutions, techniques such as skip connections between the output layer and lower layers aid gradient flow during backpropagation and ensure adequate learning.

<chunk page=449 label="Specialized ANN Architectures">
Specialized ANN architectures have been designed for specific data types: (1) Convolutional Neural Networks (CNN) for two-dimensional gridded objects such as images, extensively used in computer vision. (2) Recurrent Neural Networks (RNN) for sequential data such as speech and language. These specialized architectures incorporate domain-specific inductive biases that make them more effective than generic MLPs for their respective data types.

<chunk page=450 label="Support Vector Machines">
A support vector machine (SVM) is a discriminative classification model that learns linear or nonlinear decision boundaries to separate classes. SVM maximizes the separability of two classes and offers strong regularization capabilities to control model complexity and ensure good generalization. SVM has strong roots in statistical learning theory and is based on the principle of structural risk minimization. A unique aspect of SVM is that it represents the decision boundary using only a subset of training examples that are most difficult to classify, known as the support vectors. It is impacted only by training instances near the boundary of the two classes.

<chunk page=451 label="SVM Separating Hyperplane">
The generic equation of a separating hyperplane is w^T x + b = 0, where x represents the attributes and (w, b) represent the parameters. A data instance x_i belongs to either side of the hyperplane depending on the sign of (w^T x_i + b). If there exists a hyperplane that can perfectly separate the classes, the dataset is linearly separable. There can be infinitely many separating hyperplanes. To obtain the best generalization performance, a simple hyperplane that is robust to small perturbations is preferred, achieved using the concept of the margin of a separating hyperplane.

<chunk page=452 label="SVM Margin">
For every separating hyperplane B_i, a pair of parallel hyperplanes b_{i1} and b_{i2} touch the closest instances of both classes. These are known as the margin hyperplanes. The distance between them is the margin of the separating hyperplane. The separating hyperplane with the maximum margin is the maximum margin hyperplane. Hyperplanes with large margins tend to have better generalization performance than those with small margins because they are more robust to minor perturbations in the data. Small margin hyperplanes are more susceptible to overfitting.

<chunk page=453 label="SVM Maximum Margin Rationale">
The maximum margin hyperplane has strong foundations in statistical learning theory. The margin is inversely related to the VC-dimension of the classifier, which measures model complexity. A simpler model should be preferred over a more complex model if both show similar training performance. Hence, maximizing the margin selects the separating hyperplane with the lowest model complexity, expected to show better generalization performance. A linear SVM searches for the separating hyperplane with the largest margin, and is often known as a maximal margin classifier.

<chunk page=454 label="Linear SVM Formulation">
For a binary classification problem with n training instances where y_i in {-1, +1}, a separating hyperplane w^T x + b = 0 satisfies: w^T x_i + b > 0 if y_i = 1, and w^T x_i + b < 0 if y_i = -1. The distance of any point x from the hyperplane is D(x) = |w^T x + b| / ||w||. Let k+ and k- be the distances of the closest points from classes +1 and -1 respectively. The margin is k+ + k- = 2M. The SVM optimization problem is: max_{w,b} M subject to y_i(w^T x_i + b) >= M * ||w||. This is commonly reformulated as: min_{w,b} ||w||^2 subject to y_i(w^T x_i + b) >= 1.

<chunk page=455 label="SVM Lagrangian Primal">
The SVM constrained optimization is solved using the Lagrangian formulation: L_P = (1/2)||w||^2 - sum_{i=1}^{n} lambda_i(y_i(w^T x_i + b) - 1), where lambda_i >= 0 are the Lagrange multipliers. Taking derivatives and setting to zero: w = sum_{i=1}^{n} lambda_i y_i x_i, and sum_{i=1}^{n} lambda_i y_i = 0. This shows w can be represented entirely in terms of the Lagrange multipliers and training instances.

<chunk page=456 label="SVM Support Vectors">
From the Karush-Kuhn-Tucker (KKT) conditions: lambda_i [y_i(w^T x_i + b) - 1] = 0. This complementary slackness condition reveals that lambda_i > 0 only when x_i lies exactly on a margin hyperplane. If x_i is farther away such that y_i(w^T x_i + b) > 1, then lambda_i = 0. Training instances with lambda_i > 0 are the support vectors, the small subset closest to the separating hyperplane. Since w = sum lambda_i y_i x_i, only support vectors contribute to w. This ability to represent the decision function only in terms of support vectors gives the classifier its name.

<chunk page=457 label="SVM Dual Problem">
The dual optimization problem is: max_{lambda_i} sum_{i=1}^{n} lambda_i - (1/2) sum_{i,j} lambda_i lambda_j y_i y_j x_i^T x_j subject to sum lambda_i y_i = 0, lambda_i >= 0. Maximizing the dual is equivalent to minimizing the primal. The dual problem identifies support vectors with non-zero lambda_i values. The dual objective involves only inner products x_i^T x_j, which proves useful for learning nonlinear decision boundaries using kernel functions.

<chunk page=458 label="SVM Dual Problem Advantages">
Key advantages of the dual formulation: (1) It identifies support vectors that have non-zero Lagrange multipliers, and the solution is influenced only by instances closest to the decision boundary. This summarizes SVM learning solely in terms of support vectors. (2) The dual objective involves only inner products x_i^T x_j, enabling the use of kernel functions for nonlinear decision boundaries. Having found optimal lambda_i, w is computed from the support vectors and b = (1/n_S) sum_{i in S} -y_i w^T x_i where S is the set of support vectors. The decision function is f(x) = (sum lambda_i y_i x_i^T x) + b, and a test instance is classified by the sign of f(x).
